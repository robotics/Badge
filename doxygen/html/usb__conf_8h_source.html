124 /*******************************************************************************
125 * FIFO Size Configuration in Device mode
126 *
127 * (i) Receive data FIFO size = RAM for setup packets +
128 * OUT endpoint control information +
129 * data OUT packets + miscellaneous
130 * Space = ONE 32-bits words
131 * --> RAM for setup packets = 10 spaces
132 * (n is the nbr of CTRL EPs the device core supports)
133 * --> OUT EP CTRL info = 1 space
134 * (one space for status information written to the FIFO along with each
135 * received packet)
136 * --> data OUT packets = (Largest Packet Size / 4) + 1 spaces
137 * (MINIMUM to receive packets)
138 * --> OR data OUT packets = at least 2*(Largest Packet Size / 4) + 1 spaces
139 * (if high-bandwidth EP is enabled or multiple isochronous EPs)
140 * --> miscellaneous = 1 space per OUT EP
141 * (one space for transfer complete status information also pushed to the
142 * FIFO with each endpoint's last packet)
143 *
144 * (ii)MINIMUM RAM space required for each IN EP Tx FIFO = MAX packet size for
145 * that particular IN EP. More space allocated in the IN EP Tx FIFO results
146 * in a better performance on the USB and can hide latencies on the AHB.
147 *
148 * (iii) TXn min size = 16 words. (n : Transmit FIFO index)
149 * (iv) When a TxFIFO is not used, the Configuration should be as follows:
150 * case 1 : n > m and Txn is not used (n,m : Transmit FIFO indexes)
151 * --> Txm can use the space allocated for Txn.
152 * case2 : n < m and Txn is not used (n,m : Transmit FIFO indexes)
153 * --> Txn should be configured with the minimum space of 16 words
154 * (v) The FIFO is used optimally when used TxFIFOs are allocated in the top
155 * of the FIFO.Ex: use EP1 and EP2 as IN instead of EP1 and EP3 as IN ones.
156 * (vi) In HS case 12 FIFO locations should be reserved for internal DMA registers
157 * so total FIFO size should be 1012 Only instead of 1024
158 *******************************************************************************/
